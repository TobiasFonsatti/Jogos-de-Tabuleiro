Motivação

Como criar múltiplos jogos
sem duplicar código?

❌ Sem arquitetura
  • •Cada jogo reimplementa o loop de turnos
  • •Código de validação duplicado
  • •Adicionar um novo jogo = reescrever tudo
  • •Nenhuma padronização entre jogos
  • •Bugs se repetem em cada jogo
✅ Com POO + SOLID
  • ✦Loop único reutilizado por todos os jogos
  • ✦Tabuleiro genérico de qualquer tamanho
  • ✦Novo jogo = herdar e sobrescrever só o necessário
  • ✦Contrato claro via classe abstrata
  • ✦Testes unitários no núcleo central
💡
Solução: Criamos um núcleo genérico (src/core) com classes base que encapsulam toda a lógica compartilhada. Cada jogo herda e especializa apenas o que precisa.

Design do Sistema

Arquitetura Orientada a Objetos

🔗
Associação: Um Jogador tem uma Peca.
📦
Agregação: JogoTabuleiro recebe um Tabuleiro e lista de Jogadores.
🧬
Herança: JogoDaVelha, Lig4 e Damas herdam de JogoTabuleiro.

O núcleo em src/core é independente dos jogos. Nenhum arquivo de jogos/ é importado pelo core.

Pilar 1 de POO

Encapsulamento —
classe Tabuleiro

src/core/tabuleiro.py 
class Tabuleiro:
    def __init__(self, linhas: int, colunas: int):
        self._linhas = linhas     # privado
        self._colunas = colunas   # privado
        self._grid = [            # privado
            [None for _ in range(colunas)]
            for _ in range(linhas)
        ]

    def posicao_valida(self, linha, coluna) -> bool:
        return (0 <= linha < self._linhas and
                0 <= coluna < self._colunas)

    def is_vazio(self, linha, coluna) -> bool:
        return self._grid[linha][coluna] is None

    def posicionar_peca(self, linha, coluna, peca):
        if self.posicao_valida(linha, coluna) \
                and self.is_vazio(linha, coluna):
            self._grid[linha][coluna] = peca
            return True
        return False
🔒
Atributos com _underscore são privados por convenção. Acesso externo só via métodos públicos.
🎯
Responsabilidade Única (SRP): o Tabuleiro gerencia apenas a grade. Ele não sabe nada sobre as regras do Jogo da Velha ou do Lig-4.

Instâncias usadas

Jogo da Velha Tabuleiro(3, 3)
Lig-4 Tabuleiro(6, 7)
Damas Tabuleiro(8, 8)

Pilar 2 de POO + Padrão de Projeto

Abstração & Template Method

src/core/jogo_base.py 
from abc import ABC, abstractmethod

class JogoTabuleiro(ABC):

    @abstractmethod
    def inicializar_tabuleiro(self): ...

    @abstractmethod
    def validar_jogada(self, *args) -> bool: ...

    @abstractmethod
    def aplicar_jogada(self, *args) -> bool: ...

    @abstractmethod
    def verificar_fim_de_jogo(self) -> bool: ...

    # ← TEMPLATE METHOD: define o esqueleto do jogo
    def jogar(self):
        self.inicializar_tabuleiro()
        while self._status == "EM_ANDAMENTO":
            self._tabuleiro.exibir()
            print(f"Turno de {self.jogador_atual.nome}")
            # delega para a subclasse:
            self.aplicar_jogada(...)
        # exibe resultado final
        self._tabuleiro.exibir()
🧩
Classe Abstrata (ABC): não pode ser instanciada diretamente. Garante que todo jogo implemente os métodos obrigatórios.
📐
Template Method: o método jogar() define o fluxo geral (loop, exibição, alternância de turno). As subclasses preenchem apenas os passos específicos.

Fluxo do Template Method

① inicializar_tabuleiro() ← subclasse
② Loop enquanto EM_ANDAMENTO
③ aplicar_jogada() ← subclasse
④ verificar_fim_de_jogo() ← subclasse
⑤ Exibe resultado final

jogo.jogar()

Polimorfismo: a linha jogo.jogar() é sempre a mesma — mas o Python executa o comportamento correto de acordo com o tipo real do objeto em tempo de execução.

Pilar 4 de POO

Herança —
JogoDaVelha em detalhe

jogos/jogo_da_velha.py 
class JogoDaVelha(JogoTabuleiro):

    def __init__(self, jogador1, jogador2):
        # Herda TUDO: turnos, status, jogadores
        super().__init__(
            [jogador1, jogador2],
            Tabuleiro(3, 3)  # tabuleiro 3×3
        )

    def inicializar_tabuleiro(self):
        pass  # começa vazio

    def validar_jogada(self, linha, coluna):
        return self._tabuleiro.is_vazio(linha, coluna)

    def aplicar_jogada(self, linha, coluna):
        if self.validar_jogada(linha, coluna):
            peca = self.jogador_atual.peca
            self._tabuleiro.posicionar_peca(linha, coluna, peca)
            if self.verificar_fim_de_jogo():
                self._status = "FINALIZADO"
            else:
                self.alternar_turno()
            return True
        return False
🧬
Com super().__init__(), o Jogo da Velha herda automaticamente: alternância de turnos, propriedade jogador_atual, controle de status e o loop do Template Method.

O que a subclasse implementa

  • ✅validar_jogada: posição vazia?
  • ✅aplicar_jogada: posiciona peça e checa fim
  • ✅verificar_fim_de_jogo: linhas, colunas, diagonais e empate

Princípio Aberto/Fechado (SOLID)

Extensibilidade —
adicionar um jogo é simples

jogos/lig4.py — gravidade 
class Lig4(JogoTabuleiro):

    def __init__(self, jogador1, jogador2):
        super().__init__(
            [jogador1, jogador2],
            Tabuleiro(6, 7)  # 6 linhas, 7 colunas
        )

    # Gravidade: peça cai para linha mais baixa
    def aplicar_jogada(self, coluna: int) -> bool:
        if self.validar_jogada(coluna):
            for linha in range(5, -1, -1):
                if self._tabuleiro.is_vazio(linha, coluna):
                    self._tabuleiro.posicionar_peca(
                        linha, coluna, self.jogador_atual.peca
                    )
                    break
            ...
🔓
Open/Closed Principle: o sistema é aberto para extensão (novos jogos) e fechado para modificação (o core não precisa mudar).

Testes Unitários —
garantindo que funciona

tests/test_jogo_da_velha.py 
class TestJogoDaVelha(unittest.TestCase):

    def setUp(self):
        self.j1 = Jogador("Alice", Peca("X"))
        self.j2 = Jogador("Bob",   Peca("O"))
        self.jogo = JogoDaVelha(self.j1, self.j2)

    def test_jogada_invalida_fora_do_tabuleiro(self):
        # Posição (3,3) não existe no grid 3×3
        self.assertFalse(self.jogo.aplicar_jogada(3, 3))

    def test_jogada_invalida_posicao_ocupada(self):
        self.jogo.aplicar_jogada(0, 0)  # ocupa (0,0)
        self.assertFalse(self.jogo.aplicar_jogada(0, 0))

    def test_vitoria_linha(self):
        # Alice: (0,0), (0,1), (0,2) → vitória!
        self.jogo.aplicar_jogada(0, 0)
        self.jogo.aplicar_jogada(1, 0)
        self.jogo.aplicar_jogada(0, 1)
        self.jogo.aplicar_jogada(1, 1)
        self.jogo.aplicar_jogada(0, 2)
        self.assertEqual(self.jogo._status, "FINALIZADO")
        self.assertEqual(self.jogo._vencedor, self.j1)
